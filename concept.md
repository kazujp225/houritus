# ZettAI 破産手続支援SaaS：コンセプト定義書

---

## 1. サービス概要

### 1-1. サービス名

**ZettAI 破産手続支援SaaS**（仮称）

### 1-2. 一言定義

> 弁護士法人が提供する個人破産手続の業務を、AIで劇的に効率化するB2B業務支援SaaS

### 1-3. ポジショニング

| 項目 | 内容 |
|------|------|
| サービス形態 | B2B SaaS |
| 提供者 | ZettAI株式会社 |
| 利用者 | 弁護士法人（初期は柳田法律事務所） |
| エンドユーザー | 破産手続を希望する個人（依頼者） |
| **法律サービス提供主体** | **弁護士法人（ZettAIではない）** |

---

## 2. 解決する課題

### 2-1. 弁護士側の課題

| 課題 | 詳細 |
|------|------|
| 時間の大半が事務作業 | ヒアリング、資料整理、書類作成に膨大な時間 |
| スケールしない | 1人の弁護士が担当できる案件数に限界 |
| 定型作業の繰り返し | 破産申立書類は定型部分が多い |
| ミス・漏れのリスク | 手作業による転記ミス、確認漏れ |

### 2-2. 依頼者側の課題

| 課題 | 詳細 |
|------|------|
| 相談のハードル | 対面・電話が必要、営業時間内のみ |
| 手続の不透明さ | 何をいつまでに準備すればいいか分からない |
| 待ち時間 | 弁護士の確認待ちで進捗が止まる |

---

## 3. 提供価値

### 3-1. 弁護士法人への価値

| 価値 | 詳細 |
|------|------|
| 業務効率10倍 | 1人の弁護士が従来の10倍の案件を処理可能 |
| 品質の均一化 | AIドラフトにより書類品質が安定 |
| リスク検知 | 免責不許可事由等のリスクを自動フラグ |
| 進捗可視化 | 全案件の状況を一覧で把握 |

### 3-2. 依頼者への価値（弁護士法人経由）

| 価値 | 詳細 |
|------|------|
| 24時間相談受付 | チャットで事前ヒアリングが可能 |
| 進捗の見える化 | 必要書類・次のアクションが明確 |
| 迅速な対応 | AIアシストにより弁護士の確認が早い |

## 4. 非弁リスク回避の設計原則

### 4-1. 絶対に守る5原則

| # | 原則 | 詳細 |
|---|------|------|
| 1 | **法律サービス提供主体は弁護士法人** | 契約・窓口・請求・責任の全てが弁護士法人 |
| 2 | **AIは補助ツールに徹する** | 事実整理・ドラフト生成・進捗管理のみ |
| 3 | **対外送信は弁護士のみ** | 依頼者・債権者・裁判所への送信権限は弁護士だけ |
| 4 | **個別判断は弁護士が行う** | 「あなたは破産が最適」等の結論はAIが出さない |
| 5 | **課金は案件連動しない** | SaaS月額のみ、紹介料・成功報酬は禁止 |

### 4-2. 役割の明確な分離

```
┌─────────────────────────────────────────────────────────┐
│                    エンドユーザー（依頼者）               │
└─────────────────────────────────────────────────────────┘
                            ↓ 相談・資料提出
┌─────────────────────────────────────────────────────────┐
│              弁護士法人（柳田法律事務所）                 │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 弁護士：方針決定・承認・対外送信・最終責任        │   │
│  │ 事務職員：依頼者対応・資料収集・リマインド        │   │
│  └─────────────────────────────────────────────────┘   │
│                        ↑ 承認済み回答                    │
│                        ↓ ドラフト・フラグ                │
│  ┌─────────────────────────────────────────────────┐   │
│  │            ZettAI SaaS（業務支援ツール）          │   │
│  │  ・事前ヒアリング自動化                           │   │
│  │  ・OCR/資料抽出                                   │   │
│  │  ・書類ドラフト生成                               │   │
│  │  ・進捗管理・リマインド                           │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

## 5. ビジネスモデル

### 5-1. 収益構造

| 収益源 | 詳細 | 非弁リスク |
|--------|------|-----------|
| **月額SaaS利用料** | 席数または事務所単位の固定課金 | ✅ 安全 |
| **従量課金（案件数上限）** | 月間○件まで基本料金、超過分は追加 | ✅ 安全（成功連動でなければ） |
| ~~紹介料~~ | 1件獲得いくら | ❌ 禁止 |
| ~~成功報酬分配~~ | 回収額の○% | ❌ 禁止 |

### 5-2. 初期想定プラン

| プラン | 月額 | 含まれる内容 |
|--------|------|-------------|
| スタンダード | ¥XX万/月 | 弁護士1席、事務員2席、月間50件まで |
| プロフェッショナル | ¥XX万/月 | 弁護士3席、事務員10席、月間200件まで |
| エンタープライズ | 個別見積 | 大規模事務所向け、カスタマイズ対応 |

### 6-2. セキュリティ要件

| 項目 | 要件 |
|------|------|
| 認証 | 多要素認証（MFA）必須 |
| 通信 | TLS 1.3以上 |
| データ暗号化 | 保存時暗号化（AES-256） |
| アクセス制御 | ロールベースアクセス制御（RBAC） |
| 監査ログ | 全操作を記録、改ざん防止 |
| テナント分離 | 事務所間でデータ完全分離 |

## 9. リスクと対策

### 9-1. 法的リスク

| リスク | 対策 |
|--------|------|
| 非弁認定 | 設計原則の徹底、リーガルオピニオン取得 |
| 非弁提携認定 | 運用主体の明確化、課金モデルの適正化 |
| 個人情報漏えい | 暗号化、アクセス制御、監査ログ |

### 9-2. 技術リスク

| リスク | 対策 |
|--------|------|
| AI誤出力 | 弁護士レビュー必須、フラグ機能 |
| OCR精度不足 | 人間確認フロー、精度改善の継続 |
| システム障害 | 冗長化、バックアップ、インシデント対応手順 |

### 9-3. 事業リスク

| リスク | 対策 |
|--------|------|
| 弁護士会からの指摘 | 事前相談、透明性確保 |
| 規制変更 | 法改正のモニタリング、柔軟な設計 |
| 競合参入 | 先行者優位の確立、品質での差別化 |

## 10. 付録：用語定義

| 用語 | 定義 |
|------|------|
| 非弁行為 | 弁護士法72条に違反する、弁護士以外による法律事務の取り扱い |
| 非弁提携 | 弁護士法27条に違反する、非弁業者との提携（名義貸し・周旋受け） |
| 依頼者 | 破産手続を希望する個人（弁護士法人のクライアント） |
| エンドユーザー | 依頼者と同義（本サービスでは弁護士法人ではなく依頼者を指す） |
| 内部用ドラフト | AIが生成した書類案。弁護士レビュー前は依頼者に非表示 |
| 要確認フラグ | 弁護士による追加確認が必要な項目のマーク |
| 承認ログ | 弁護士がいつ・何を・どのように承認したかの記録 |
| 送信ゲート | 弁護士アカウントのみが対外送信できる権限制御 |

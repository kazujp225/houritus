## 2. ZettAI ⇔ 弁護士法人：SaaS利用契約

### 2-1. 基本条項

```
第1条（目的）
本契約は、ZettAI株式会社（以下「甲」という）が提供する破産手続支援SaaS
（以下「本サービス」という）を、弁護士法人●●（以下「乙」という）が利用
することに関し、必要な事項を定めることを目的とする。

第2条（サービス内容）
1. 甲は、乙に対し、以下の機能を含む本サービスを提供する。
   (1) 依頼者向けオンライン相談受付機能
   (2) 資料アップロード・OCR処理機能
   (3) AIによる書類ドラフト生成機能
   (4) 弁護士向けレビュー・承認機能
   (5) 進捗管理・リマインド機能
   (6) 監査ログ機能
2. 本サービスは、乙が行う法律事務の補助ツールとして提供されるものであり、
   甲が法律事務を行うものではない。
```

### 2-2. 役割分担条項（非弁対策の核心）

```
第3条（役割分担）
1. 乙は、本サービスを利用する法律事務の提供主体であり、依頼者との契約関係、
   法的判断、対外的な送信・提出の責任を負う。
2. 甲は、本サービスのシステム提供者として、技術的な保守・運用を行う。
3. 甲は、以下の行為を行わない。
   (1) 依頼者に対する法的助言・回答の提供
   (2) 依頼者に対する個別案件の方針提示
   (3) 債権者・裁判所への書面の送信・提出
   (4) 依頼者との委任契約の締結
4. 甲が生成するAIドラフトは、乙の弁護士が確認・承認するまで依頼者に
   表示されないものとする。

第4条（送信権限）
1. 本サービスにおける依頼者・債権者・裁判所への送信機能は、乙の弁護士
   アカウントのみが実行可能な設計とする。
2. 甲のシステム管理者および乙の事務職員アカウントからは、前項の送信を
   実行することができない。
```

### 2-3. 料金条項（非弁対策：案件連動禁止）

```
第5条（料金）
1. 乙は、甲に対し、本サービスの利用料として、別紙に定める月額利用料を
   支払う。
2. 前項の利用料は、以下の基準に基づき算定される。
   (1) 基本料金（月額固定）
   (2) ユーザーアカウント数に応じた従量課金
   (3) オプション機能の利用料
3. 利用料は、個別案件の受任数、成功報酬、回収額等に連動しないものとする。
4. 甲は、乙から以下の名目による金員を受領しない。
   (1) 案件紹介料
   (2) 成功報酬の分配
   (3) 1件あたりの課金
```

### 2-4. データ・知的財産条項

```
第6条（データの帰属）
1. 乙が本サービスに入力・アップロードしたデータ（依頼者情報、案件情報、
   書類等）の所有権および権利は、乙に帰属する。
2. 甲は、前項のデータを、本サービスの提供および改善の目的以外に使用しない。
3. 甲は、乙のデータを第三者に開示しない。ただし、法令に基づく場合、または
   乙の事前の書面による承諾がある場合はこの限りでない。

第7条（AIモデル）
1. 甲が本サービスで使用するAIモデルの学習には、乙のデータを使用しない。
2. 甲は、乙のデータを外部LLMプロバイダに送信する場合、個人を特定できる
   情報をマスキング処理した上で送信する。
```

### 2-5. 秘密保持条項

```
第8条（秘密保持）
1. 甲および乙は、本契約に関連して知り得た相手方の秘密情報を、第三者に
   開示または漏洩してはならない。
2. 甲は、乙の依頼者に関する情報を、弁護士法上の守秘義務に準じて取り扱う。
3. 本条の義務は、本契約終了後も5年間存続する。
```

### 2-6. 契約期間・解約条項

```
第9条（契約期間）
1. 本契約の有効期間は、契約締結日から1年間とする。
2. 期間満了の1ヶ月前までに、いずれの当事者からも書面による解約の申し出が
   ない場合、本契約は同一条件で1年間自動更新されるものとし、以後も同様とする。

第10条（解約）
1. いずれの当事者も、1ヶ月前までに書面で通知することにより、本契約を
   解約することができる。
2. 契約終了時、甲は乙の指示に従い、乙のデータを返却または削除する。
3. 甲は、データの削除を完了した場合、乙に対し削除証明書を発行する。
```

## 3. ZettAI ⇔ 弁護士法人：個人情報の取扱いに関する覚書

### 3-1. 基本条項

```
第1条（目的）
本覚書は、SaaS利用契約に基づき甲が乙から委託を受けて取り扱う個人情報
（以下「委託データ」という）の取扱いについて、必要な事項を定める。

第2条（委託データの範囲）
委託データには、以下の情報が含まれる。
(1) 依頼者の氏名、住所、連絡先
(2) 依頼者の金融情報（借入額、資産状況等）
(3) 債権者情報
(4) 案件に関する書類データ
(5) 通信記録

第3条（管理責任）
1. 委託データの管理責任者は乙とする。
2. 甲は、乙の委託先として、乙の指示に従い委託データを取り扱う。
```

### 3-2. 安全管理措置

```
第4条（安全管理措置）
甲は、委託データについて、以下の安全管理措置を講じる。
(1) 組織的安全管理措置
    ・個人情報保護責任者の設置
    ・取扱規程の整備
    ・従業者への教育
(2) 物理的安全管理措置
    ・サーバーのアクセス制限
    ・入退室管理
(3) 技術的安全管理措置
    ・アクセス制御
    ・通信の暗号化（TLS 1.3以上）
    ・保存時の暗号化（AES-256）
    ・アクセスログの記録
(4) 人的安全管理措置
    ・従業者との秘密保持契約
    ・アクセス権限の最小化
```

### 3-3. アクセス制限

```
第5条（アクセス制限）
1. 甲の従業者は、原則として委託データの内容にアクセスしない。
2. 障害対応等のためアクセスが必要な場合、甲は以下の手順に従う。
   (1) 乙に対しアクセス申請（理由・範囲を明記）
   (2) 乙による承認
   (3) アクセスログの記録
   (4) アクセス完了後、乙への報告
3. 甲が依頼者に直接連絡する場合、以下に限定する。
   (1) ログイン方法等の技術的案内
   (2) システム障害のお知らせ
   (3) その他、法的内容を含まない事務連絡
```

### 3-4. 再委託

```
第6条（再委託）
1. 甲が委託データの取扱いを第三者に再委託する場合、事前に乙の書面による
   承諾を得る。
2. 再委託先には、本覚書と同等の義務を課す。
3. 甲は、再委託先の行為について、乙に対し責任を負う。
4. 本覚書締結時点で乙が承諾する再委託先は、別紙に定める。
   ・クラウドインフラ：AWS（Amazon Web Services）
   ・LLMプロバイダ：Anthropic（Claude API）
```

### 3-5. 事故対応

```
第7条（事故対応）
1. 甲は、委託データの漏洩、滅失、毀損等の事故が発生した場合、または発生の
   おそれがある場合、直ちに乙に報告する。
2. 甲は、乙の指示に従い、原因調査、被害拡大防止、再発防止措置を講じる。
3. 甲は、事故の経緯および対策を書面で乙に報告する。
```

### 3-6. 監査

```
第8条（監査）
1. 乙は、甲の委託データの取扱状況について、年1回を限度として監査を実施
   することができる。
2. 甲は、監査に必要な情報提供および立入りに協力する。
3. 監査の結果、改善が必要と認められた場合、甲は速やかに是正措置を講じる。
```
